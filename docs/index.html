Learnings

[pattern] The Done-gate and `alreadyOnBoard` pre-filter are mutually exclusive: a board item with `linkedIssueNumber === N` prevents issue N from ever reaching LLM triage, so the gate serves as pure defense against LLM hallucination (claiming `already_done` based on unlinked Done items). [anti-pattern] Adding a board item to make a test "pass the gate" inadvertently triggers the `alreadyOnBoard` path instead — test authors must reason about both filters together. [domain] When a new guard invalidates existing behavior-locking tests, the correct response is to update the tests to assert the new (correct) behavior, not work around the guard. [tool-usage] Reading the full test file before making test changes avoids misaligned mock sequences when function internals gain extra calls.
